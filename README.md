# Pujukan

Pujukan is a bilingual React + Vite storefront for premium local meats. The final site is a routed web app with a preserved homepage, dedicated content pages, and a Supabase-backed product catalog.

## Final Experience

- A homepage that keeps the original hero section and summary sections for products, recipes, about, and contact
- Dedicated routes for `/products`, `/recipes`, `/about`, and `/contact`
- English and Korean support through `react-i18next`
- Product cards loaded from Supabase
- Product tags built from `product_type` and `meat_type`
- Tag-based filtering in the product gallery
- Contact details displayed on both the homepage and the dedicated contact page

## Product Data

The product gallery reads from a Supabase table with these columns:

- `product_id`
- `korean_name`
- `english_name`
- `product_type`
- `meat_type`

`korean_name` is shown when the app language is Korean, and `english_name` is shown when the app language is English. `product_type` and `meat_type` are rendered as filterable tags.

## Supabase Setup

Use a public read-only policy so the website can fetch products without allowing edits from the client.

```sql
create table if not exists public.products (
  product_id bigint generated by default as identity primary key,
  korean_name text not null,
  english_name text not null,
  product_type text,
  meat_type text
);

alter table public.products enable row level security;

drop policy if exists "public read products" on public.products;

create policy "public read products"
on public.products
for select
using (true);
```

If the table already exists, you only need the RLS and policy statements.

## Environment Variables

Create a `.env` file at the project root:

```env
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-key
VITE_SUPABASE_PRODUCTS_TABLE=products
```

`VITE_SUPABASE_PRODUCTS_TABLE` is optional, but it must match the exact table name if you are not using the default `products`.

## Local Development

```bash
npm install
npm run dev
```

The app runs locally with Vite, usually at `http://localhost:5173`.

```bash
npm run build
npm run lint
```

## Notes

- The homepage still renders on `/`.
- Navbar links use client-side routing instead of hash anchors.
- The product gallery shows status messages when Supabase is not configured, when fetches fail, or when filters return no matches.
